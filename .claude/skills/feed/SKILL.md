# /feed — 지식 주입(트레이닝) 워크플로

사용자가 유튜브 링크를 주면, 이 저장소의 사주 지식베이스(dosa-app/kb)에 대해
"없는 정보인가"를 판정하고, 가치가 있으면 주입한 뒤, 무엇이 보완됐는지 보고한다.

## 절차 (링크마다 반복)

1. **판정**: `python3 dosa-app/kb-tools/check_new_video.py "<URL>"`
   - 출력: 다루는 주제(기존 커버리지 대비) / 얇은 키 적중 / 사전에 없는 용어 후보 / 수집 가치 판정
2. **주입 결정**: 판정 '중간' 이상이거나 사용자가 `--ingest`를 붙였으면
   `python3 dosa-app/kb-tools/check_new_video.py "<URL>" --save` 로 자막을 fulltext/에 저장.
   '낮음(중복)'이면 저장하지 않고 그 사실만 보고.
3. **색인 갱신** (저장분이 1개라도 있으면 — 기계산출물이므로 손편집 금지, 스크립트 재실행만):
   ```
   python3 dosa-app/kb-tools/yt_filter.py
   python3 dosa-app/kb-tools/scan_interactions.py
   python3 dosa-app/kb-tools/scan_impressions.py
   ```
4. **신규 용어 처리**: 판정에서 '사전에 없는 용어'가 나왔으면 —
   - 명리 용어가 맞으면 `scan_interactions.py`(PATTERNS/SINSAL 부류) 또는 `scan_impressions.py`(ELEMENT_TERMS)에
     패턴을 **코드로** 추가하고 재실행 (지식 사전의 확장 = 코드 수정).
   - 애매하면 사용자에게 "이 용어를 사전에 등재할까?"로 확인.
5. **보완 보고** (스캔 전후 대조):
   | 링크 | 채널 | 판정 | 메운 키(전→후 문단수) | 신규 용어 |
   + "남은 공백" 목록(커버리지 5건 미만 키) 갱신.
6. **커밋·푸시**: 갱신된 색인(json)·저장 자막 등록·패턴 코드 변경을 현재 작업 브랜치에 커밋.
   커밋 메시지: `지식 주입: <영상 제목 요약> — <메운 키들>`.
7. 증류까지 원하면(사용자가 요청 시): 해당 키 번들 재생성(`prepare_interaction_bundles.py`) 후
   증류 에이전트 투입 — DISTILL_GUIDE.md 규칙 준수(축자 인용·관점차이 병기).

## 원칙

- **주입 여부의 최종 기준은 신규성**: 이미 두터운 주제의 중복 영상은 정중히 반려하고 이유(기존 N문단)를 보여줄 것.
- 무속·타 역학 관점 영상은 배제하지 말고 `folk_etc` 등급으로 주입 (관점차이 병기용 — yt_filter.py가 자동 처리).
- 자막이 안 받아지는 영상(멤버십·자막 없음)은 그 사실을 보고하고, 사용자가 텍스트를 직접 붙여넣으면 수동 등록 경로 안내.
- 개인정보(상담 실명 등)가 들어간 영상은 주입 전에 사용자에게 알릴 것.
