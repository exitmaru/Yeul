유튜브 자막(글자) 추출 — 도구 제작 완료 추출은 내 PC에서

260712 · 보내준 유튜브 주소 321개(+재생목록 4개) · 요청: “글자 다 추출해서 영상별로 저장 + 워드 파일”

321고유 영상 (중복 제거 전 600+줄)
4재생목록 (자동 펼침)
0이 서버에서 추출 가능 IP 차단
1실행 스크립트 (검증 완료)

핵심: 왜 여기서 자막이 안 뽑혔나

결론부터 — 이 클라우드 서버 IP가 유튜브에 봇으로 찍혀 차단됐어. 코드 문제가 아니라 네트워크(주소) 차단이야.

  • youtube-transcript-api → RequestBlocked (IP 차단 명시)
  • yt-dlp → “Sign in to confirm you're not a bot”
  • watch 페이지 직접 요청 → 302 리다이렉트 → google.com/sorry (구글의 “비정상 트래픽” 차단 페이지). 이게 결정적 증거.

데이터센터(클라우드) IP는 유튜브가 자동요청을 막지만, 집·회사 일반 인터넷에서는 대부분 정상 동작해.

그래서 만든 것: 내 PC에서 한 번에 돌아가는 도구

레포에 저장함 → tools/youtube_transcripts/

파일역할
extract.py주소 읽기 → 영상별 자막 추출 → txt 저장 → 워드 1개 생성 (실패영상도 안전 처리)
youtube_urls.txt네가 보낸 321영상+4재생목록 (중복 제거 완료) — 그대로 쓰면 됨
requirements.txt필요한 부품 3개 목록
README.md따라 하기 설명서 (처음이어도 되게 한 단계씩)

검증 완료(오프라인): 입력 파싱(321개 정확) · 워드 생성(유효한 .docx) · 영상별 txt · 실패영상 표기 · 요약 CSV — 전 과정 정상. 네트워크 자막요청만 이 서버에서 막힘.

내 PC에서 실행하는 법 (3줄)

  1. tools/youtube_transcripts 폴더로 이동
  2. pip install -r requirements.txt
  3. python extract.py

끝나면 output/유튜브_자막_모음.docx(전부 묶은 워드) + output/텍스트/0001_….txt(영상별) + output/결과요약.csv가 생겨. 자세한 건 README.md.

전 · 후 · 리스크 · 장단점

전(Before)

주소 600+줄이 뒤섞여 있고(중복 다수), 자막 텍스트는 하나도 없음.

후(After)

321개로 정리 + 재현 가능한 추출 도구. 내 PC에서 실행하면 영상별 txt + 워드 완성.

리스크 / 한계
  • 자막이 꺼진 영상은 어디서 돌려도 못 가져옴 → CSV에 ‘실패’로 표기하고 계속 진행.
  • 한꺼번에 많이 요청하면 일반 IP도 일시 차단될 수 있어 → 영상 사이 1초 쉼(조절 가능).
  • 자동생성 자막은 오탈자·띄어쓰기 오류가 있을 수 있음(유튜브 원본 한계).
대안

내 PC가 어려우면, 브라우저 쿠키를 내보내 클라우드에서 우회 실행하는 방법도 있음(민감정보라 번거로움). 필요하면 안내할게.
