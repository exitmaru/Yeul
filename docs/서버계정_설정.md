# 서버 계정 설정 — Cloudflare에서 딱 한 번 누르면 끝 (260713)

> **뭘 만든 건가**: 만세 앱에 **이메일+비밀번호 가입/로그인**이 생겼다. 출생 정보가 서버(Cloudflare D1 = 클라우드에 있는 작은 데이터베이스)에 저장돼서 **폰을 바꿔도 계정으로 이어진다**.
> **코드는 전부 레포에 들어 있고**, 남은 건 Cloudflare 대시보드에서 데이터베이스를 만들어 연결하는 것뿐 — 아래 4단계, 전부 클릭이다. (이건 계정 권한이 필요해서 운영자만 할 수 있는 부분)

## 구조 한 줄

```
브라우저(만세 앱) ──> /api/auth/*, /api/profile   ← functions/ 폴더 (Pages Functions = 서버 코드)
                                │
                                └──> D1 데이터베이스 "DB" (users · sessions · profiles 3테이블)
```

- 비밀번호는 **원문 저장 안 함** — PBKDF2-SHA256(솔트+6만 회 반복)으로 해시만 저장.
- 세션 토큰도 원문은 브라우저 쿠키에만, DB엔 SHA-256 해시만(DB가 새어도 위조 불가).
- 외부 서비스 의존 0 — 전부 Cloudflare 안에서 끝.

## 설정 4단계 (처음이어도 5분 · 단계마다 바로 가는 URL)

1. **D1 만들기** → https://dash.cloudflare.com/b3ca893503b580ac6afba9a7b284d93f/workers/d1
   **Create Database** → 이름 `saju-users` → Create.
2. **테이블 만들기**: 위 페이지에서 방금 만든 `saju-users` 클릭 → **Console** 탭 → 아래 스키마 원문을 통째로 복사해 붙여넣고 **Execute**. ("Success" 비슷한 메시지가 나오면 성공)
   스키마 원문(복사용) → https://raw.githubusercontent.com/muteno/Saju/claude/consolidate-repo-drafts-lkt6pk/functions/schema.sql
3. **Pages에 연결(바인딩)** → https://dash.cloudflare.com/b3ca893503b580ac6afba9a7b284d93f/pages/view/saju/settings
   **바인딩(Bindings) 섹션 → 추가 → D1 데이터베이스** → 변수 이름 = `DB` (대문자, 정확히 이 이름) · 데이터베이스 = `saju-users` 선택 → 저장. **환경(프로덕션/미리보기) 선택이 있으면 둘 다.**
4. **재배포** → https://dash.cloudflare.com/b3ca893503b580ac6afba9a7b284d93f/pages/view/saju
   최신 배포의 **⋯ → 재배포(Retry deployment)** (바인딩은 새 배포부터 적용).

> 테스트 주소: PR #32 머지 전까지는 브랜치 프리뷰 https://claude-consolidate-repo-draf.saju-6w4.pages.dev/app/ 에서 가입 흐름을 확인한다(머지 후엔 본 배포 URL).

## 확인하는 법 (1분)

- 배포 URL 열기 → 첫 진입에 **"간단 가입" 시트**가 올라온다 → 이메일·비밀번호(8자+)·부를 이름 넣고 **가입하고 시작** → 출생 정보 입력 → 저장하고 보기.
- 시크릿 창(또는 다른 기기)에서 다시 열어 **로그인** → 방금 저장한 출생 정보가 자동으로 뜨면 성공.
- 설정 전이라면? 앱은 죽지 않고 **구경 모드**로 열리며, 가입을 눌러도 "서버 준비 전(D1 미연결)" 안내만 뜬다.

## API 한눈에 (참고)

| 메서드·경로 | 하는 일 | 비고 |
|---|---|---|
| POST `/api/auth/signup` | 가입 {email, password, name} | 성공 시 바로 로그인(쿠키) |
| POST `/api/auth/login` | 로그인 {email, password} | 실패 문구는 일부러 두루뭉술(계정 털이 방지) |
| POST `/api/auth/logout` | 로그아웃 | 세션 삭제 + 쿠키 만료 |
| GET `/api/auth/me` | 내 정보 + 저장된 출생 정보 | 미로그인 = 401 |
| GET/PUT `/api/profile` | 출생 정보 조회/저장 | 시간 미상 허용(time_unknown) |

## 알아둘 것 · 다음

- **세션 30일** 유지(쿠키 HttpOnly·Secure). 비밀번호 재설정(이메일 발송)은 다음 단계 — 메일 발송 서비스 연결이 필요해서 이번엔 뺐다.
- 요청 속도 제한(무차별 로그인 시도 방어)은 Cloudflare 대시보드 **Security → WAF → Rate limiting rules**에서 `/api/auth/*` 대상으로 걸어두면 좋다(선택).
- 이 계정 테이블이 앞으로 **꼬미 미연시의 관계 저장소**(LV·호칭·침전물) 자리다 — profiles 옆에 relationship 테이블을 붙이는 게 다음 확장.
